cppBuildEnv
===========
# ディレクトリ構成
## レベル1
* Main
実行ファイルを作成するソースコードを格納するディレクトリ
* Lib
共通ライブラリを作成するソースコードを格納するディレクトリ


## レベル2
* bin
ビルドによる生成物を格納するディレクトリ
* src
ソースコードを格納するディレクトリ
* test
テストコードを格納するディレクトリ
* obj
ビルドによる中間生成物を格納するディレクトリ
* count
規模測定の結果等を格納するディレクトリ
* gcov
カバレッジ測定結果等を格納するディレクトリ
* html
Doxygenによる自動生成ドキュメントを格納するディレクトリ

# ビルド手順
## ビルド
(あらかじめsrcディレクトリへ移動する）
% make

## テスト実行（Google Test Frameworkが必要）
（あらかじめtestディレクトリへ移動する）
% make
% make run

# コマンド
* run
binディレクトリの実行ファイル（TARGET)を実行する
* debug
デバッグ構成でのビルド実行
* release
リリース構成でのビルド実行
* clean
ビルドの生成物、中間生成物を削除
* count
規模測定を実行する（CodeCountが必要）
* tag
タグ情報を生成する（ctagが必要）
* dox
doxygenの実行（doxygenが必要）
レベル1のディレクトリ内にあるDoxyfileの設定を使用する
* indent
ソースコードの整形を行う（uncrustifyが必要）
レベル1ディレクトリと同じレベルにあるuncrustify.cfgの設定を使用する

# プロジェクトごとに必要な設定
buildディレクトリ内のMakefile、src,testディレクトリにあるMakefileで
プロジェクト毎の設定とコメントしてあるもの（コンパイラのフラグなど）
はプロダクトに合わせて修正してください。

# 補足
基本的には生産物のソースコードをsrc、テストコードをtestに置いて
それぞれのディレクトリでmakeコマンドを使うという想定です。
